Under the hood

Built on Apple's own crypto

▹

Vault is an AES-256 encrypted sparsebundle created via Apple's hdiutil — sparse, so it only uses the disk space it actually holds.

▹

Your master password is passed to hdiutil over stdin — never via command-line args, never written to disk.

▹

Unlocked, it mounts at /Volumes/CustodiaVault, hidden from the Finder sidebar. Locked, only an opaque blob remains.

▹

Back up to a single compressed, encrypted .dmg protected by your same password — restore by double-clicking it, no Custodia required.

First launch

Install in 30 seconds

1

Unzip & move to Applications

Double-click the downloaded .zip, then drag Custodia.app into your Applications folder.

2

Open it the first time

Custodia is a free, independently-built app and isn't signed with a paid Apple Developer certificate, so macOS asks you to confirm on first launch. Right-click (or Control-click) Custodia → Open → Open. You only do this once.

3

Set your master password

Pick a password of at least 8 characters. This unlocks everything in your vault.

"Apple could not verify Custodia"?

On recent macOS, if right-click → Open doesn't show an Open button, go to:

System Settings → Privacy & Security, scroll to the Security section, and click "Open Anyway" next to the Custodia message.

Advanced users can instead clear the quarantine flag in Terminal:

xattr -dr com.apple.quarantine /Applications/Custodia.app

No password recovery

Your master password is the only key — there is no reset, no backdoor, no "forgot password." If you lose it, the vault's contents are unrecoverable. That's the point. Keep it somewhere safe.
